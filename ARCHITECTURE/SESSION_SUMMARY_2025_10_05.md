### ✅ Решена проблема холодного старта стрима

**Проблема:**
- Первые 10-30 секунд после старта: буферизация, обрывы, зависания
- Нужно было перезагружать страницу несколько раз
- После прогрева всё стабилизировалось

**Причина:**
- `hls_window = 6` (мы думали "6 сегментов", на деле "6 секунд")
- Результат: 6 сек / 2 сек = **3 сегмента** (слишком мало!)
- API показывал isLive слишком рано (при 4 сегментах)

**Решение (НА СЕРВЕРЕ):**
```bash
# /etc/srs/srs.conf
hls_window      20;           # Было: 6 → Стало: 20 (10 сегментов)
hls_wait_keyframe off;        # Было: on → мгновенный старт
```

**Решение (В КОДЕ):**
- Увеличили буферы HLS.js: 30-60 секунд
- Добавили UI "Стрим запускается..." во время прогрева
- Порог isWarmingUp: 4 сегмента (временно, можно увеличить до 8)

**Результат:**
✅ Плейлист накапливает 10+ сегментов  
✅ Прогрев ~15 секунд (было 30+)  
✅ Меньше обрывов при старте

### ✅ Решена проблема "чёрный экран" на десктопе

**Проблема:**
- Видео НЕ работало на десктопе (Chrome/Firefox)
- Видео РАБОТАЛО на мобилке (iOS/Android)
- Чёрный экран, только аудио

**Диагностика (логи):**
```
❌ [mp4-remuxer]: No keyframe found out of 50 video samples
❌ [stream-controller]: Found no media in fragment
❌ [buffer-controller]: Fragments must have at least one ElementaryStreamType
❌ Unsupported transition between "audio" and "audio,video" SourceBuffers
```

**Причина:**
- Кодировщик **Apple VT H264** создавал поток БЕЗ ключевых кадров
- HLS.js (десктоп) требует keyframes → отбрасывал все кадры
- Нативный плеер (мобилка) более терпим → работал с тем что есть

**Решение:**
```
OBS → Настройки → Вывод → Вещание:

Кодировщик: x264 (было: Apple VT H264)
Интервал ключевых кадров: 0s (auto)
Профиль: baseline
Предустановка: veryfast
```

**Результат:**
✅ Видео ЗАРАБОТАЛО на десктопе!  
⚠️ Но СЛОМАЛОСЬ аудио (нужно разобраться)

### 1. Аудио не работает
**Симптомы:** После переключения на x264 аудио сломалось

**План диагностики:**
- Проверить логи `[AUDIO DEBUG]` в консоли
- Посмотреть что в `/audio` странице
- Проверить Icecast (если используется)

---

### 2. Холодный старт всё ещё требует перезагрузок
**Симптомы:** Нужно 2-3 раза перезагрузить страницу для стабилизации

**Возможные причины:**
- Дырки в буфере (buffer holes)  
- Overlapping fragments (2019ms overlap)
- Первый keyframe не на границе сегмента

**План решения:**
- Попробовать keyframe interval = 50 (вместо 0s)
- Проверить логи FFmpeg на сервере
- Возможно нужно подкрутить транскодинг

---

### 3. Оптимизация настроек
**После стабилизации:**
- Убрать debug логи (или сделать опциональными)
- Довести isWarmingUp до 8 сегментов (сейчас 4)

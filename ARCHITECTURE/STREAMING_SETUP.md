# Настройка стриминга на сервере

## Статус настройки ✅

Сервер полностью настроен и готов к работе со стримингом.

## Настройки для OBS Studio

**Основной стрим:**
- Сервер: `rtmp://185.200.178.73/live`
- Ключ потока: `ваш_ключ` (например: `main`, `conference`, `test`)

**Конференции:**
- Сервер: `rtmp://185.200.178.73/conf`
- Ключ потока: `ваш_ключ`

## URL для просмотра

**HLS потоки:**
- Основной: `http://185.200.178.73/live/ваш_ключ.m3u8`
- Конференции: `http://185.200.178.73/live/ваш_ключ.m3u8`

## Архивы

Все стримы автоматически записываются в:
- Путь: `/srv/streaming/archive/`
- Формат: FLV файлы с уникальными именами

## Управление сервисами

```bash
# Перезапуск nginx
sudo systemctl restart nginx

# Проверка статуса
sudo systemctl status nginx

# Просмотр логов
sudo tail -f /var/log/nginx/error.log
```

## Тестирование

Для тестирования можно использовать ffmpeg:

```bash
ffmpeg -f lavfi -i testsrc=duration=10:size=1280x720:rate=30 \
       -f lavfi -i sine=frequency=1000:duration=10 \
       -c:v libx264 -preset veryfast -c:a aac \
       -f flv rtmp://185.200.178.73/live/test
```

## Структура файлов

```
/srv/streaming/
├── live/           # HLS потоки
│   └── ключ/       # Папка для каждого потока
│       ├── index.m3u8
│       └── *.ts    # Сегменты видео
└── archive/        # Записи стримов
    └── *.flv       # Архивные файлы
```

## Интеграция с приложением

Ваше приложение уже настроено для работы со стримингом:

1. **StreamLinkBlock** компонент получает активную ссылку через API `/api/stream-link`
2. **API endpoint** `/api/stream-link` управляет ссылками на стримы
3. **База данных** содержит таблицу `StreamLink` для хранения ссылок

## Безопасность

- RTMP сервер доступен только на порту 1935
- HLS потоки доступны через HTTP на порту 80
- Архивы защищены правами доступа www-data

**Статус:** Рабочая версия с ручным запуском FFmpeg HLS

---

## 📋 Архитектура системы

```
OBS (720p @ 1500 Kbps)
  ↓ RTMP rtmp://spoken-word.ru/live/main
NGINX RTMP :1935
  ├─→ YouTube Forward: 720p (copy, оригинал) ✅ автозапуск
  ├─→ RuTube Forward: 720p (copy, оригинал) ✅ автозапуск
  ├─→ Icecast Audio: MP3 @ 128k ✅ автозапуск
  └─→ FFmpeg HLS: 360p @ 600k ⚠️ РУЧНОЙ запуск
       ↓
     /srv/streaming/hls/main/index.m3u8
       ↓
     Пользователи сайта
```

---

## ✅ Что работает автоматически

При запуске трансляции из OBS **автоматически** запускаются:

1. ✅ **Icecast аудио стрим** (MP3 @ 128k)
2. ✅ **Forward на YouTube** (оригинальное качество 720p)
3. ✅ **Forward на RuTube** (оригинальное качество 720p)

---

## ⚠️ Что нужно запускать ВРУЧНУЮ

**FFmpeg HLS транскодирование** для видео на сайте:

```bash
npm run hls:start
```

Это создаёт **360p @ 600 Kbps** поток для мобильных и десктопов.

---

## 🚀 Процесс запуска трансляции

### 1. Запусти OBS

**Настройки в OBS (если ещё не настроено):**
- Сервер: `rtmp://spoken-word.ru/live`
- Ключ: `main`
- Битрейт: 1500 Kbps
- Разрешение: 1280x720
- FPS: 25
- Keyframe Interval: 2 секунды
- Профиль: baseline

Нажми **"Начать трансляцию"**

### 2. Подожди 5 секунд

Дай NGINX запустить все автоматические скрипты.

### 3. Запусти FFmpeg HLS вручную

```bash
npm run hls:start
```

Или если на сервере:
```bash
ssh amster "bash /usr/local/bin/start-hls-transcoding.sh main"
```

### 4. Проверь что всё работает

```bash
npm run hls:status
```

Должно показать запущенный процесс FFmpeg.

### 5. Открой сайт

**https://spoken-word.ru/live** - должно показывать видео (360p)

## 📊 Текущие параметры стрима

### Для сайта (FFmpeg HLS):
- **Разрешение:** 640x360
- **Битрейт видео:** 600 Kbps
- **Битрейт аудио:** 96 Kbps (AAC)
- **Профиль:** H.264 Baseline
- **Сегменты:** 2 секунды
- **Плейлист:** 30 сегментов (60 сек буфера)
- **Без B-frames** для стабильности

### Для YouTube/RuTube:
- **Разрешение:** 1280x720 (оригинал)
- **Битрейт:** 1500 Kbps (оригинал)
- **Copy** (без перекодирования)

### Для Icecast (аудио):
- **Формат:** MP3
- **Битрейт:** 128 Kbps
- **Частота:** 44100 Hz

## 🐛 Известные проблемы

### Проблема 1: FFmpeg HLS не запускается автоматически

**Симптом:** После запуска OBS видео не появляется на сайте.

**Решение:** Запустить вручную `npm run hls:start`

**Причина:** Конфликт в NGINX между `push` и `exec_publish` (будет исправлено завтра)

### Проблема 2: Нужно создавать директорию вручную

**Симптом:** FFmpeg HLS запускается, но файлы не создаются.

**Решение:** 
```bash
ssh amster "mkdir -p /srv/streaming/hls/main && chown www-data:www-data /srv/streaming/hls/main"
```

Или проще: `npm run fix`

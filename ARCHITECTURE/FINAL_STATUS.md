## ⚠️ ПРОБЛЕМА: Сжатие увеличивает размер

**Причина:** Видео уже сжато кодеком H.264!  
Повторное сжатие ухудшает качество и УВЕЛИЧИВАЕТ размер.

**Решение (сделано):**
- Проверяем кодек перед обработкой
- Если H.264/H.265 → копируем `-c:v copy` (без пересжатия)
- Быстрее (в 5-10 раз) и без увеличения размера!

## 🎯 КАК ТЕПЕРЬ РАБОТАЕТ:

### Загрузка файла:
```
1. Админ выбирает 1 файл (multiple убран)
2. Next.js сохраняет в temp (может крашнуться на 500MB+)
3. Добавляет в Redis очередь
4. Возвращает jobId
```

### Обработка worker:
```
1. Worker берёт задачу из очереди (concurrency: 1)
2. Проверяет кодек (ffprobe)
3. Если H.264/H.265 → копирует (-c:v copy) ⚡ быстро
4. Если другой → сжимает (scale 720p, crf 28) 🐢 медленно
5. Сохраняет результат
6. Обновляет БД
7. Чистит temp
8. Обновляет статус в Redis
```

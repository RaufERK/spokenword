План и стратегия:

1. Мы обновились на nginx 1.28.0 (stable) для сайта (HTTP/HTTPS, HLS).
   RTMP в этой версии больше не нужен внутри nginx.
2. RTMP мы выносим в отдельный сервер — будем использовать SRS (Simple Realtime Server) или MediaMTX.
   Задача: настроить OBS → RTMP → SRS, а уже SRS будет отдавать HLS/WebRTC,
   которые nginx 1.28 раздаёт пользователям через HTTPS.
3. nginx 1.28 должен остаться лёгким и стабильным: только reverse proxy, HTTPS, HLS-раздача, без RTMP-модуля.
4. Основная цель: исправить проблемы со стримингом на мобильных устройствах.
   Значит, нужно организовать выдачу HLS (и желательно WebRTC для минимальной задержки).
5. Нам нужен пошаговый план установки и интеграции:

   - Установка SRS (или MediaMTX) как отдельного процесса/сервиса.
   - Конфиг OBS для стрима в RTMP-вход SRS.
   - Конфиг SRS для отдачи HLS/WebRTC.
   - Конфиг nginx 1.28 для раздачи HLS пользователям по HTTPS.
   - Проверка работы на мобильных устройствах.
6. Сразу учесть systemd-юниты, автозапуск, и понятную структуру конфигов.
   Всё должно работать устойчиво: OBS → RTMP → SRS → HLS/WebRTC → nginx → зрители.

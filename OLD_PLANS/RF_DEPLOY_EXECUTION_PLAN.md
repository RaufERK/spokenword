## Риски

### 1. Nginx на РФ пока пустой

Сейчас там только `default` site. Даже если приложение заведется, домен не заработает без отдельного server block.

### 2. На РФ не хватает зависимостей рантайма

Без `redis-server` не взлетят очередь и upload/compression flow.  
Без `ffmpeg` не взлетит обработка видео.

### 3. PM2/Node завязаны на nvm

Команды через SSH без `bash -lc` или без загрузки `nvm` могут давать ложное ощущение, что `node` и `pm2` отсутствуют.

### 4. Нельзя копировать весь EU-сервер как есть

Если перенести все подряд, можно утащить:

- старые `.next`
- битые symlink
- мусорные логи
- лишние `node_modules`
- старое PM2 state

Правильнее переносить код отдельно, а данные отдельно.

### 5. Absolute URL и будущий режим `eu`/`www`

В коде уже есть несколько жестких URL на основной домен. Для первого запуска на РФ это не блокер, но для схемы `www + eu` это нужно будет дочистить.

### 6. Cookie / auth между доменами

Перед запуском схемы `www` + `eu` нужно отдельно проверить cookie domain и поведение NextAuth. Иначе возможны проблемы с логином между доменами.

### 7. SQLite остается временным компромиссом

Для текущего быстрого переезда это нормальный путь, но нужно помнить, что write-master должен быть только один. Во время тестов на РФ нельзя допустить, чтобы обе площадки начали жить как две независимые prod-базы.

## Правило переключения

Пока РФ не пройдет smoke test, `www.spoken-word.ru` остается на Амстердаме.

Только после этого:

1. заморозить записи на EU
2. сделать финальную копию `data.db`
3. сделать финальную дельту по медиа
4. переключить DNS
5. пройти повторный smoke test

# 🎬 Исправление сбоев в начале стрима

**Дата:** 2025-10-04  
**Проблема:** Сбои видео/аудио в первые секунды стрима, затем стабилизация  
**Статус:** ✅ Исправлено

---

## 🔍 Анализ проблемы

### Почему происходили сбои:

1. **SRS генерирует сегменты с нуля** - при старте стрима создается `main-0.ts`, `main-1.ts` и т.д.
2. **Период нестабильности 3-5 секунд** - пока не накопится минимум 3-4 сегмента (6-8 секунд видео)
3. **Плееры подключались слишком рано** - до того, как появились все необходимые сегменты
4. **HLS требует минимум 3 сегмента** - для стабильной работы
5. **Старые сегменты могли оставаться** - `hls_dispose=3` секунды

---

## ✅ Что было исправлено

### 1. Улучшен API `/api/stream-status` ✅

**Файл:** `app/api/stream-status/route.ts`

**Новые параметры в ответе:**
- `isWarmingUp` - true если стрим только начался (< 4 сегментов)
- `segmentCount` - количество сегментов в плейлисте
- `streamAge` - примерный возраст стрима в секундах
- `tsFilesOnDisk` - количество .ts файлов на диске

**Пример ответа:**
```json
{
  "isLive": true,
  "streamKey": "main",
  "lastModified": "2025-10-04T10:30:00.000Z",
  "fileAge": 2,
  "isWarmingUp": true,
  "segmentCount": 2,
  "streamAge": 4,
  "tsFilesOnDisk": 2
}
```

## 🎯 Как это работает теперь

### Сценарий 1: Начало стрима

1. **OBS запускает трансляцию** → RTMP идет в SRS
2. **SRS создает первый сегмент** `main-0.ts`
3. **API обнаруживает стрим** `isLive: true, isWarmingUp: true`
4. **Страница показывает "Стрим запускается..."** 🎬
5. **Ждем 6 секунд** (3 сегмента по 2 сек)
6. **SRS успевает создать 3-4 сегмента** ✅
7. **Плеер загружается** с уже стабильным потоком
8. **Видео начинается плавно** без сбоев! 🎉

### Сценарий 2: Подключение к работающему стриму

1. **Пользователь открывает страницу**
2. **API возвращает** `isLive: true, isWarmingUp: false, streamAge: 120`
3. **Плеер загружается сразу** - стрим уже стабилен
4. **Стандартная retry логика** (3 попытки)

### Сценарий 3: Ошибка в первые 30 секунд

1. **Плеер получает ошибку** (404 на сегмент)
2. **Определяет "молодой" стрим** `isYoungStream: true`
3. **Использует 8 попыток** с задержкой 1.5 сек
4. **Показывает "Подключение к стриму..."** вместо страшных ошибок
5. **Успешно подключается** через 2-3 попытки ✅

## 🎓 Почему эти значения?

### Warm-up период 6 секунд (видео):
- SRS создает сегмент каждые 2 секунды
- За 6 секунд = 3 полных сегмента
- HLS требует минимум 3 сегмента для стабильности
- + небольшой запас на задержки сети

### Warm-up период 5 секунд (аудио):
- Аудио сегменты меньше по размеру
- Быстрее создаются и загружаются
- 5 секунд = 2-3 сегмента достаточно

### 8 попыток для видео:
- Более тяжелые данные
- Больше вероятность задержек
- 8 попыток × 1.5 сек = до 12 секунд на подключение

### 10 попыток для аудио:
- Аудио критичнее для трансляций
- Меньший размер данных = быстрее retry
- 10 попыток × 1.2 сек = до 12 секунд

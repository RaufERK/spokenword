### 2. **Upload Microservice** (порт 3006)

**Статус:** 🟢 Online (uptime 4 дня)

```
PM2 процесс: spokenword-upload
PID: 1806572
Memory: 66.2 MB
Version: Express.js 4.18.2
```

**Зачем:**
Next.js имеет жёсткий лимит 10MB на `req.formData()`, который **невозможно обойти**. Поэтому создан отдельный Express микросервис.

**Что делает:**
- Принимает загрузку файлов до **5GB** (через `busboy`)
- Обрабатывает multipart/form-data в streaming режиме
- Добавляет задачи в BullMQ очередь для сжатия
- Проверяет видео кодеки через FFprobe

**Endpoints:**
- `POST /upload/conference` — загрузка в архив конференций
- `POST /upload/packages` — загрузка платного контента
- `POST /test/upload` — тестовый endpoint (без авторизации)
- `GET /job-status/:jobId` — статус задачи сжатия
- `GET /health` — health check

**Nginx routing:**
```nginx
# Nginx перенаправляет напрямую на 3006
/api/conf-archive/upload → http://127.0.0.1:3006/upload/conference
/api/admin/packages/upload → http://127.0.0.1:3006/upload/packages
```

### 3. **Compression Worker** (фоновый процесс)

**Статус:** 🟢 Online (uptime 4 дня)

```
PM2 процесс: spokenword-compression-worker
PID: 1806542
Memory: 64.1 MB
```

**Что делает:**
- Обрабатывает очередь BullMQ (`video-compression`)
- Сжимает видео через FFmpeg
- Умная логика кодеков:
  - **H.264/HEVC** → копирование с оптимизацией (`-c:v copy`)
  - **Другие** → перекодирование в H.264 (libx264, CRF 28, 720p)
- Обновляет базу данных после сжатия

**Конфигурация:**
- Worker script: `upload-service/workers/compression-worker.ts`
- Лимит памяти: 2GB
- Использует Redis для очереди
- Автоматический retry при ошибках

```
Браузер (<audio> tag, нативно)
```

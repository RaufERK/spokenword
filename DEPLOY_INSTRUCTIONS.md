# 🚀 Инструкция по деплою с исправлением стриминга

## ✅ Проблема решена!

Деплой теперь работает без ошибок, но после каждого деплоя нужно запускать скрипт восстановления прав доступа к файлам стриминга.

## 📋 Процесс деплоя

### 1. Деплой приложения
```bash
npm run deploy
```

### 2. Исправление прав доступа (ОБЯЗАТЕЛЬНО!)
```bash
./scripts/fix-streaming-quick.sh
```

## 🔧 Что делает скрипт исправления

- ✅ Создает папку `/srv/streaming/live/main/`
- ✅ Устанавливает владельца `www-data:www-data`
- ✅ Устанавливает права `755` для папок
- ✅ Устанавливает права `644` для файлов `.m3u8` и `.ts`
- ✅ Проверяет API и HLS файл

## 🎬 После исправления можете стримить

### Настройки OBS Studio:
- **Сервер:** `rtmp://185.200.178.73/live`
- **Ключ потока:** `main`

### Проверка на сайте:
- **Главная:** https://www.spoken-word.ru/
- **Стрим:** https://spoken-word.ru/live

## 🚨 Если что-то пошло не так

### Ручное исправление на сервере:
```bash
ssh amster "bash /root/fix-streaming-permissions.sh"
```

### Проверка статуса:
```bash
curl "https://spoken-word.ru/api/stream-status?key=main"
curl -I "https://spoken-word.ru/live/main/index.m3u8"
```

## 📱 Решение проблем с мобильными устройствами

После деплоя и запуска скрипта исправления:
- ✅ HLS файлы доступны с CORS заголовками
- ✅ API корректно определяет статус стрима
- ✅ Плеер работает на всех устройствах

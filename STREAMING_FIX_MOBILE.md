# 📱 Исправление мобильной трансляции

## ✅ Что было сделано

### 1. Проблема
- Видео не работало на мобильных устройствах
- Плейлист ссылался на несуществующие сегменты (404 ошибки)
- После перезагрузки страницы трансляция падала
- Длинные сегменты (10 сек) и большое окно (60 сек) создавали задержки

### 2. Решение (ФИНАЛЬНОЕ)
- **Используем встроенный HLS nginx-rtmp** с оптимизированными параметрами
- **Короткие сегменты:** 2 секунды (быстрый старт)
- **Короткое окно:** 12 секунд (6 сегментов, нет битых ссылок)
- **Автоочистка:** старые сегменты удаляются сразу
- **Исправили пути:** используем `/live/main.m3u8` напрямую

### 3. Изменения в коде

#### Frontend
- `app/live/page.tsx`: URL изменён на `https://spoken-word.ru/live/main.m3u8`
- `components/HlsPlayer.tsx`: 
  - Добавлен старт у live-edge (`startPosition: -3`)
  - Уменьшен буфер для мобильных (`maxBufferLength: 10`)
  - Добавлен автоплей после загрузки

#### Backend (на сервере)
- Nginx RTMP с встроенным HLS и оптимизированными параметрами
- `hls_fragment 2s` - короткие сегменты для быстрого старта
- `hls_playlist_length 12s` - маленькое окно предотвращает 404
- `hls_cleanup on` - автоматическое удаление старых файлов

## 🔧 Технические детали

### HLS параметры (nginx-rtmp)
- **Сегменты:** 2 секунды (`hls_fragment 2s`)
- **Окно плейлиста:** 12 секунд (`hls_playlist_length 12s`)
- **Непрерывность:** `hls_continuous on`
- **Автоочистка:** `hls_cleanup on`
- **Без вложенных папок:** `hls_nested off`

### Почему это работает
1. **Короткие сегменты (2с)** = быстрый старт на мобильных, меньше задержка
2. **Маленькое окно (12с)** = только свежие сегменты, нет 404 ошибок
3. **Встроенный HLS nginx-rtmp** = стабильная генерация без внешних процессов
4. **Автоочистка** = старые файлы удаляются сразу, нет накопления

## ⚠️ Важно

# 📱 Исправление проблем с мобильными устройствами

## 🎯 Проблема была найдена и решена!

**Причина:** Nginx создавал HLS файлы в корне папки `/srv/streaming/live/` (например, `main.m3u8`), но API искал их в подпапке `/srv/streaming/live/main/index.m3u8`.

## ✅ Что было исправлено:

### 1. **Конфигурация nginx RTMP**
- Изменено `hls_nested on` на `hls_nested off`
- Теперь nginx создает файлы в правильной структуре

### 2. **Символические ссылки**
- Создается ссылка `main.m3u8 -> main/index.m3u8`
- Создаются ссылки на все сегменты `.ts`

### 3. **Автоматическое исправление**
- Скрипт `fix-streaming-permissions.sh` теперь автоматически создает все необходимые ссылки

## 🚀 Процесс деплоя (без изменений):

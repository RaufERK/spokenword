## Основные сущности системы

### 1. Пользователи (Users)
- Регистрация и авторизация через NextAuth
- Роли: USER, ADMIN, SUPER
- После авторизации пользователи попадают на страницу со ссылками:
  - **Оплатившие**: доступ к трансляции учебного класса
  - **Неоплатившие**: доступ только к открытым мероприятиям

### 2. Мероприятия (Events)
Ключевая сущность системы. Пользователи оплачивают доступ к конкретным мероприятиям.

#### Типы мероприятий (5 в год):
1. **Осенняя конференция** (онлайн)
2. **Зимняя конференция** (онлайн)
3. **Весенняя конференция** (онлайн)
4. **Летняя конференция** (онлайн)
5. **Весенний класс** (очное мероприятие, часто совпадает с весенней конференцией)

**Примечание**: Даты мероприятий меняются каждый раз, они не фиксированные.

#### Поля мероприятия:
- ID
- Название (например, "Весенняя конференция 2026")
- Тип (конференция/класс)
- Формат (онлайн/очное)
- Дата начала
- Статус (предстоящее/активное/завершенное)
- Дата создания

### 3. Оплаты (Payments)
Связывают пользователя с мероприятием.

#### Поля оплаты:
- ID пользователя
- ID мероприятия
- Дата оплаты
- Статус (активна/отозвана)

**Важно**: У каждого пользователя должна быть **история оплат** - список всех мероприятий, которые он когда-либо оплачивал.

### 4. Видео архив
Каждое видео в архиве привязано к конкретному мероприятию.

#### Поля видео:
- ID
- Название
- Ссылка на видео
- ID мероприятия (к какому мероприятию относится)
- Дата загрузки

## Логика работы системы

### Регистрация мероприятия
1. За месяц или за пару недель до мероприятия администратор создает запись о нем на странице "Мероприятия"
2. Указывается дата начала мероприятия
3. Мероприятие становится доступным для оплаты

### Процесс оплаты
1. Пользователь оплачивает участие в конкретном предстоящем мероприятии
2. Администратор отмечает оплату в системе, привязывая ее к:
   - Пользователю
   - Конкретному мероприятию
   - Дате оплаты
3. Если это новый пользователь (первая оплата), дата мероприятия используется по умолчанию

### Доступ к архиву
**Правило**: Пользователь имеет доступ к архиву того мероприятия, которое он оплатил, в течение **1 месяца** с даты оплаты.

Пример:
- Пользователь оплатил "Весеннюю конференцию 2026" 15 марта 2026
- Он получает доступ ко всем видео, помеченным как "Весенняя конференция 2026"
- Доступ действует до 15 апреля 2026

### Загрузка видео в архив
При загрузке видео администратор обязательно указывает:
- К какому мероприятию относится видео (тег/метаинформация)
- Это позволяет системе показывать видео только тем пользователям, кто оплатил это мероприятие

## Интерфейсы

### Пользовательский интерфейс
- Страница после авторизации со ссылками на:
  - Текущую трансляцию (для оплативших)
  - Открытые мероприятия (для всех)
- Личный кабинет с историей оплат
- Доступ к архиву оплаченных мероприятий
- Чат с функционалом:
  - Лента сообщений
  - Реакции
  - Два поля ввода (текст и ссылки)

### Админская панель
Красно-розовое меню с разделами:

#### 1. Пользователи
- Управление пользователями
- Таблица со всеми пользователями (~300 человек)
- Фильтры:
  - Все
  - С оплатой
  - Без оплаты
  - Новые (у кого вообще нет ни одной оплаты)
  - Админы
- Массовые операции:
  - Дать доступ (активировать оплату)
  - Отозвать доступ
- Привязка оплаты к конкретному мероприятию
- История оплат каждого пользователя

#### 2. Ссылки на конференцию
- Управление ссылками на текущие трансляции
- Разделение на:
  - Ссылки для оплативших
  - Ссылки на открытые мероприятия

#### 3. Загрузка в Архив
- Загрузка видео
- **Обязательное указание мероприятия** при загрузке
- Привязка видео к конкретному мероприятию

#### 4. Мероприятия (новый раздел)
- Список всех мероприятий (предстоящих, текущих, завершенных)
- Создание нового мероприятия:
  - Название
  - Тип (конференция/класс)
  - Формат (онлайн/очное)
  - Дата начала
- Просмотр статистики по каждому мероприятию:
  - Количество оплативших
  - Список оплативших пользователей
  - Количество видео в архиве

#### 5. Платные пакеты
- Управление платными материалами
- Доступ к специальным курсам и семинарам

#### 6. Чат (модерация)
- Удаление сообщений (для админов и модераторов)
- Просмотр всех сообщений

### Переключение режимов
- Кнопка "АДМИН" в пользовательском интерфейсе
- Кнопка "Вид пользователя" в админке
- Позволяет администраторам быстро переключаться между режимами

## Состояния пользователя

### Новый пользователь
- Зарегистрирован, но ни разу не оплачивал
- Доступ только к открытым мероприятиям

### Пользователь с активной оплатой
- Оплатил конкретное мероприятие
- Оплата действительна (менее месяца с даты оплаты)
- Имеет доступ к:
  - Трансляциям текущего мероприятия
  - Архиву оплаченного мероприятия

### Пользователь с истекшей оплатой
- Оплата была более месяца назад
- Доступ к архиву закрыт
- Сохраняется история оплат

## Бизнес-логика

### Определение активной оплаты
Оплата считается активной, если:
1. Дата оплаты была менее 1 месяца назад
2. Оплата не была отозвана администратором

### Доступ к контенту
Пользователь видит контент (видео), если:
1. Видео привязано к мероприятию, которое он оплатил
2. Его оплата этого мероприятия активна (менее месяца)

### Массовые операции
Администратор может:
1. Выбрать несколько пользователей галочками
2. Массово дать доступ к конкретному мероприятию
3. Массово отозвать доступ

## Детали реализации функций

### Активация оплаты пользователю
При нажатии кнопки "+ОПЛАТА" в таблице пользователей:
1. Открывается модальное окно
2. В модалке есть селект (выпадающий список) со всеми доступными мероприятиями
3. Администратор выбирает мероприятие, за которое пользователь оплатил
4. При подтверждении:
   - Создается запись оплаты с привязкой к пользователю и мероприятию
   - Устанавливается дата оплаты (текущая дата)
   - Пользователь получает доступ к архиву этого мероприятия на 1 месяц

### Отзыв оплаты
Возможны два способа отзыва оплаты:
1. **Индивидуальный** - кнопка "Отозвать" у каждого пользователя с активной оплатой
2. **Массовый** - выбор нескольких пользователей галочками и массовый отзыв через кнопку "Отозвать доступ"

При отзыве оплаты:
- Доступ к архиву немедленно блокируется
- Запись об оплате остается в истории, но помечается как "отозвана"

### Загрузка видео в архив
На странице "Загрузка в Архив":
1. Администратор загружает видео
2. **Обязательное поле**: Выбор мероприятия из селекта
3. Видео привязывается к выбранному мероприятию
4. Только пользователи, оплатившие это мероприятие, смогут увидеть это видео

### Просмотр профиля пользователя
При нажатии кнопки "Профиль" в таблице пользователей:
1. Система генерирует две ссылки на профиль:
   - Ссылка для доступа из РФ
   - Ссылка для доступа из-за пределов РФ
2. Обе ссылки автоматически копируются в буфер обмена
3. Показывается модальное окно с подтверждением: "Ссылки на профиль скопированы"
4. В модалке кнопка "ОК" для закрытия

### Создание и редактирование мероприятия
При создании/редактировании мероприятия:
1. Название мероприятия (текстовое поле)
2. Тип: Конференция или Класс (селект)
3. Формат: Онлайн или Очное (селект)
4. **Дата начала** - используется датапикер (выбор даты из календаря)
5. Валидация: все поля обязательны

После создания мероприятие:
- Появляется в списке на странице "Мероприятия"
- Становится доступным для выбора при активации оплаты
- Становится доступным для выбора при загрузке видео

## Важные замечания

1. **История оплат** - критически важна. Каждая оплата сохраняется навсегда, даже если доступ истек.
2. **Привязка к мероприятию** - всегда указывается при оплате и при загрузке видео.
3. **Автоматическая проверка** - система должна автоматически проверять срок действия оплаты (1 месяц).
4. **Гибкость дат** - даты мероприятий не фиксированные, каждый раз задаются вручную.
5. **Модерация чата** - только админы и модераторы могут удалять сообщения.
6. **Копирование ссылок профиля** - при нажатии "Профиль" генерируются и копируются две ссылки (для РФ и вне РФ).
7. **Обязательный выбор мероприятия** - при активации оплаты и загрузке видео выбор мероприятия обязателен.

## Система чата

### Типы чатов

#### 1. Общий чат (📢)
- Виден всем пользователям
- Все могут писать и видеть сообщения
- Админы могут удалять сообщения
- При клике на имя пользователя - открывается личный чат с ним

#### 2. Поддержка (🛟)
- Канал обращений к администрации
- Пользователи пишут сюда по проблемам доступа
- **Для обычных пользователей**: видят только свою переписку с админами
- **Для админов**: видят ВСЕ обращения всех пользователей (список диалогов)
- Админ выбирает обращение и отвечает конкретному пользователю

#### 3. Личные чаты (💬)
- Создаются при клике на имя пользователя в общем чате
- Видны только двум участникам переписки
- Переписка один-на-один
- Возможность обмениваться контактами добровольно

### Интерфейс чата

### Функции чата

#### Общий чат
- Все сообщения видны всем
- Реакции на сообщения (👍❤️😊🔥👏)
- Возможность прикреплять ссылки
- Клик на имя автора → открывает личный чат
- Админы могут удалять любые сообщения

#### Чат поддержки
- Пользователь создаёт обращение
- Все админы видят новые обращения
- Админ отвечает - пользователь видит ответ
- История обращений сохраняется
- Бейджи с количеством непрочитанных

#### Личные чаты
- Один-на-один переписка
- Полная приватность
- История сохраняется
- Индикаторы непрочитанных сообщений
- Возможность делиться контактами (email, телефон) по желанию

### Непрочитанные сообщения
- Красный бейдж с цифрой у чата
- Счётчик непрочитанных в списке чатов
- При открытии чата - сообщения помечаются прочитанными

### Создание личного чата
1. Пользователь видит сообщение в общем чате
2. Кликает на имя автора
3. Открывается/создается личный чат
4. Чат добавляется в список чатов

### Для администраторов
**В поддержке:**
- Список всех обращений пользователей
- Непрочитанные выделены
- Клик на обращение → открывает переписку
- Возможность отвечать и решать проблемы

**В общем чате:**
- Модерация (удаление сообщений)
- Закрепление важных сообщений
- Отправка объявлений

# 🎵 Миграция аудио-страницы на HLS

## 📋 Описание задачи

**Проблема:**
- `/live` (видео) работает через SRS HLS ✅
- `/audio` не работает — пытается получить поток от Icecast, но тот пустой ❌
- Причина: nginx-rtmp выключен → `exec_publish` не срабатывает → ffmpeg не запускается → Icecast не получает поток

**Решение:**
- Переключить `/audio` на HLS от SRS (как `/live`)
- Использовать audio-only плеер с тем же HLS потоком
- Отключить Icecast (больше не нужен)

**Целевая аудитория:** мобильные браузеры 2020+ (iOS 13+, Android 8+)

**Ожидаемая задержка:** ~3-4 секунды (оптимизированный HLS)

## 📝 Технические детали

### Текущая схема (НЕ РАБОТАЕТ):
```
OBS → RTMP → SRS → HLS (видео) → /live ✅
              ↓
        (nginx-rtmp ВЫКЛЮЧЕН)
              ↓
        ffmpeg НЕ ЗАПУСКАЕТСЯ
              ↓
        Icecast ПУСТОЙ → /audio ❌
```

### Новая схема (БУДЕТ РАБОТАТЬ):
```
OBS → RTMP → SRS → HLS (видео+аудио)
                      ↓
                      ├─→ /live (video+audio) ✅
                      └─→ /audio (audio only) ✅
```

### HLS настройки для аудио:
- **Fragment:** 1 секунда (быстрый старт)
- **Window:** 4 сегмента (минимальная задержка)
- **Задержка:** ~3-4 секунды
- **Формат:** AAC 128kbps (уже в потоке от OBS)
